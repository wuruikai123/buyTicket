# 特殊票券核销流程完整说明

## 功能概述

特殊票券系统支持完整的生成、核销、记录查询流程。B端生成的二维码可以在C端扫码核销，核销记录会实时同步到B端。

## 完整流程

### 1. B端生成票券和二维码

**操作步骤：**
1. 登录B端管理系统
2. 进入"门票管理" -> "特殊票券管理"
3. 点击"批量生成票券"，输入数量（如5000）
4. 等待生成完成
5. 在列表中点击任意票券的"查看详情"
6. 查看并下载二维码（PNG格式）

**二维码内容：**
- 纯文本格式：票券编号（如：ST00000001）
- 尺寸：300x300像素
- 可打印或电子方式分发

### 2. C端扫码核销

**操作步骤：**
1. 登录C端核销系统
2. 点击"扫码核销"
3. 使用摄像头扫描票券二维码
4. 系统自动识别票券类型：
   - ST开头：特殊票券
   - T开头：普通订单
5. 调用统一核销接口
6. 显示核销结果

**核销逻辑：**
```
扫描二维码 
  ↓
提取票券编号（ST00000001）
  ↓
判断类型（ST开头=特殊票券）
  ↓
调用统一核销接口 /api/v1/admin/verify/scan
  ↓
后端验证：
  - 票券是否存在
  - 状态是否为未使用
  ↓
更新票券状态为已使用
记录核销时间和核销人
  ↓
返回核销成功
```

### 3. B端查看核销记录

**方式一：特殊票券列表筛选**
1. 进入"门票管理" -> "特殊票券管理"
2. 在状态筛选中选择"已使用"
3. 查看所有已核销的票券
4. 列表显示：
   - 票券编号
   - 核销时间（绿色高亮）
   - 核销人（蓝色高亮）
   - 创建时间

**方式二：查看详情**
1. 点击任意票券的"查看详情"
2. 详情对话框显示完整信息：
   - 票券编号
   - 状态（未使用/已使用）
   - 创建时间
   - 核销时间
   - 核销人
   - 备注
   - 二维码

## 技术实现

### 后端统一核销接口

**接口地址：** `POST /api/v1/admin/verify/scan`

**请求参数：**
```json
{
  "code": "ST00000001",           // 票券编号或订单号
  "adminId": 1,                   // 核销管理员ID（可选）
  "adminName": "张三"              // 核销管理员姓名
}
```

**响应示例（成功）：**
```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "type": "special",            // 类型：special=特殊票券，normal=普通订单
    "ticketCode": "ST00000001",
    "verifyTime": "2026-03-13 20:30:15",
    "message": "特殊票券核销成功"
  }
}
```

**响应示例（失败）：**
```json
{
  "code": -1,
  "msg": "票券已使用",
  "data": null
}
```

### 前端C端核销逻辑

**文件：** `frontend-c/src/utils/orders.ts`

```typescript
export async function verifyOrder(code: string): Promise<boolean> {
  // 获取当前登录管理员信息
  const adminInfo = getAdminInfo()
  
  // 调用统一核销接口
  await request.post('/admin/verify/scan', {
    code: code.trim(),
    adminId: adminInfo.adminId,
    adminName: adminInfo.adminName
  })
  
  return true
}
```

**文件：** `frontend-c/src/views/ScanVerify.vue`

```typescript
// 扫码后处理
const onDecode = async (result: string) => {
  // 提取编号（支持JSON或纯文本）
  let code = extractCode(result)
  
  // 验证格式（T或ST开头）
  if (!code.startsWith('T') && !code.startsWith('ST')) {
    // 无效格式
    return
  }
  
  // 调用核销
  await verifyOrder(code)
  
  // 显示结果
  showResult(code)
}
```

## 核销规则

### 1. 状态验证
- 只有状态为"未使用"（status=0）的票券才能核销
- 已使用的票券再次扫码会提示"票券已使用"

### 2. 唯一性保证
- 每个票券编号唯一
- 一个票券只能核销一次
- 核销后状态不可逆

### 3. 核销人记录
- 记录核销管理员ID和姓名
- 从C端登录信息中获取
- 如果未登录，显示为"核销员"

### 4. 时间记录
- 核销时间精确到秒
- 使用服务器时间
- 不可修改

## 错误处理

### 常见错误及解决方案

| 错误信息 | 原因 | 解决方案 |
|---------|------|---------|
| 票券不存在 | 票券编号错误或未生成 | 检查编号是否正确 |
| 票券已使用 | 票券已经核销过 | 提示用户该票券已使用 |
| 无效的二维码格式 | 二维码内容不符合规范 | 确保二维码包含正确的票券编号 |
| 核销失败 | 网络或服务器错误 | 检查网络连接，重试 |

## 常见问题

**Q: 二维码可以重复使用吗？**
A: 不可以。每个票券只能核销一次，核销后状态变为已使用，不能再次核销。

**Q: 如果误核销了怎么办？**
A: 目前不支持撤销核销。建议后续添加核销撤销功能，仅限管理员操作。

**Q: C端扫码需要联网吗？**
A: 需要。核销操作需要实时调用后端接口验证和更新状态。

**Q: B端可以手动核销吗？**
A: 目前只支持C端扫码核销。如需手动核销，可以在B端输入票券编号调用核销接口。

**Q: 核销记录可以导出吗？**
A: 可以。在特殊票券列表中，筛选"已使用"状态，然后点击"导出票券"按钮。

**Q: 如何批量打印二维码？**
A: 导出票券列表后，使用Python脚本或在线工具批量生成二维码图片，然后批量打印。
