# 特殊票券功能说明

## 功能概述

特殊票券系统用于生成永久有效的票券，供甲方使用。这些票券不绑定用户、日期或时间段，可以随时核销，核销后状态变为已使用。

## 数据库表结构

### special_ticket 表

```sql
CREATE TABLE special_ticket (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  ticket_code VARCHAR(50) UNIQUE NOT NULL COMMENT '票券编号（唯一）',
  qr_code_url VARCHAR(500) COMMENT '二维码图片URL',
  status TINYINT DEFAULT 0 COMMENT '状态：0=未使用，1=已使用',
  verify_time DATETIME COMMENT '核销时间',
  verify_admin_id BIGINT COMMENT '核销管理员ID',
  verify_admin_name VARCHAR(100) COMMENT '核销管理员姓名',
  create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
  update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  remark VARCHAR(500) COMMENT '备注'
);
```

## 票券编号格式

- 格式：`ST + 8位数字`
- 示例：ST00000001, ST00000002, ST00005000
- 前缀 ST 表示 Special Ticket（特殊票券）

## 后端接口

### 管理端接口

#### 1. 批量生成票券
- **接口**：`POST /api/v1/admin/special-ticket/batch-generate`
- **参数**：count（生成数量，默认5000，最大10000）
- **说明**：批量生成特殊票券，自动分配唯一编号

#### 2. 查询票券列表
- **接口**：`GET /api/v1/admin/special-ticket/list`
- **参数**：
  - page: 页码
  - size: 每页数量
  - status: 状态筛选（0=未使用，1=已使用）
  - ticketCode: 票券编号模糊查询
- **说明**：分页查询特殊票券列表

#### 3. 获取统计信息
- **接口**：`GET /api/v1/admin/special-ticket/statistics`
- **返回**：
  - total: 总票券数
  - unused: 未使用数量
  - used: 已使用数量

#### 4. 导出票券
- **接口**：`GET /api/v1/admin/special-ticket/export`
- **参数**：status（可选，筛选状态）
- **说明**：导出所有票券编号，用于生成二维码

#### 5. 删除票券
- **接口**：`DELETE /api/v1/admin/special-ticket/{id}`
- **说明**：删除未使用的票券（已使用的不能删除）

### 核销接口

#### 统一核销接口
- **接口**：`POST /api/v1/admin/verify/scan`
- **参数**：
  - code: 核销码（订单号或票券编号）
  - adminId: 管理员ID
  - adminName: 管理员姓名
- **说明**：自动识别是普通订单还是特殊票券
  - 以 ST 开头：特殊票券
  - 其他：普通订单号

#### 查询核销码信息
- **接口**：`GET /api/v1/admin/verify/query`
- **参数**：code（核销码）
- **说明**：查询核销码信息，不执行核销操作

## 前端功能

### B端管理页面

路径：`/ticket/special`

**功能：**
1. 统计信息展示（总数、未使用、已使用）
2. 批量生成票券
3. 票券列表查询（支持状态筛选、编号搜索）
4. 导出票券列表（CSV格式）
5. 删除未使用的票券

**操作流程：**
1. 点击"批量生成票券"按钮
2. 输入生成数量（建议5000张）
3. 确认生成
4. 等待生成完成
5. 点击"导出票券"按钮，下载CSV文件
6. 使用CSV文件中的票券编号生成二维码

### C端核销功能

**更新说明：**
- 核销功能已支持特殊票券
- 扫描二维码后自动识别票券类型
- 特殊票券核销成功后状态变为已使用

## 二维码生成

### 方案一：在线生成工具
使用在线二维码生成工具，批量生成二维码：
- 输入票券编号（如：ST00000001）
- 生成二维码图片
- 批量下载

### 方案二：程序批量生成
使用Python或其他语言编写脚本：

```python
import qrcode
import csv

# 读取CSV文件
with open('tickets.csv', 'r') as f:
    reader = csv.reader(f)
    next(reader)  # 跳过表头
    
    for row in reader:
        ticket_code = row[0]
        
        # 生成二维码
        qr = qrcode.QRCode(version=1, box_size=10, border=5)
        qr.add_data(ticket_code)
        qr.make(fit=True)
        
        img = qr.make_image(fill_color="black", back_color="white")
        img.save(f'qrcodes/{ticket_code}.png')
```

## 使用流程

### 管理员操作流程

1. **生成票券**
   - 登录B端管理系统
   - 进入"门票管理" -> "特殊票券管理"
   - 点击"批量生成票券"
   - 输入数量（如5000）
   - 确认生成

2. **导出票券**
   - 点击"导出票券"按钮
   - 下载CSV文件
   - 文件包含所有票券编号

3. **生成二维码**
   - 使用票券编号生成二维码
   - 可以使用在线工具或编写脚本批量生成
   - 打印或分发二维码

4. **核销票券**
   - 使用C端核销系统扫描二维码
   - 系统自动识别为特殊票券
   - 核销成功后状态变为已使用

### 用户使用流程

1. 获得特殊票券二维码
2. 到展馆入口出示二维码
3. 工作人员使用C端系统扫描
4. 核销成功，允许入场
5. 该票券变为已使用状态，不能再次使用

## 注意事项

1. **票券编号唯一性**：系统自动确保每个票券编号唯一
2. **一次性使用**：每个票券只能核销一次
3. **永久有效**：特殊票券没有时间限制
4. **不可恢复**：已使用的票券不能重置为未使用
5. **删除限制**：只能删除未使用的票券
6. **批量生成限制**：单次最多生成10000张

## 数据统计

系统提供实时统计：
- 总票券数：已生成的所有票券
- 未使用数：可以使用的票券
- 已使用数：已经核销的票券

## 安全性

1. 票券编号唯一，不会重复
2. 核销需要管理员权限
3. 记录核销时间和核销人
4. 已使用的票券不能再次核销
5. 所有操作都有日志记录
